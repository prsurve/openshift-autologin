Auto-Login Behaviour
🤖 Auto-detect & Login
Auto-fill when OpenShift login page detected
💬 Show confirmation prompt
Ask before auto-filling credentials
🆕 Open in new tab
Open cluster URL in a new tab when logging in
🔙 Open tabs in background
Keep current tab active when opening multiple clusters
⏱️ Sequential login (1s delay)
Open clusters one by one instead of all at once

Auto-detect works by checking if the current page is an OpenShift login page and matching the URL to one of your saved clusters. If a match is found, credentials are filled automatically (or after confirmation).

🔒 SSL Certificate Warnings: Chrome does not allow extensions to bypass certificate warnings automatically (security restriction). You must manually click Advanced → Proceed.

⚡ WORKAROUND - Accept certificates first:
1. Before adding a cluster, manually visit its URL in a new tab
2. Click "Advanced" → "Proceed to [site] (unsafe)"
3. Chrome remembers this choice and won't show the warning again
4. Now add the cluster - auto-login will work without prompts!

Alternative: Use http:// URLs if your cluster supports it (not recommended for production).
